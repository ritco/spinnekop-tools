  <action>
Creeer `scripts/deploy/locatie-scanner/install-service.ps1` — een PowerShell script dat de user via RDP op de server draait.

Het script doet het volgende (in volgorde):

1. **Variabelen bovenaan** (makkelijk aanpasbaar):
   ```powershell
   $ServiceName = "LocatieScanner"
   $InstallDir = "C:\LocatieScanner"
   $PythonExe = "C:\Users\ridderadmin\AppData\Local\Programs\Python\Python312\python.exe"
   $NssmExe = "$InstallDir\nssm.exe"
   $Port = 5050
   ```
   BELANGRIJK: Python pad is waar Python 3.12 staat op de server. Als het pad anders is, moet de user het aanpassen.

2. **Check of NSSM aanwezig is** in `$InstallDir`. Zo niet: download nssm 2.24 van https://nssm.cc/release/nssm-2.24.zip, extract `win64/nssm.exe` naar `$InstallDir`. Als download niet lukt (geen internet op server), toon duidelijke melding dat user `nssm.exe` handmatig moet plaatsen.

3. **Stop bestaande service** als die al draait (`nssm stop $ServiceName` met error silencing).

4. **Kopieer bestanden**:
   - Kopieer `locatie_scanner.py` van deploy-bron naar `$InstallDir\locatie_scanner.py`
   - Het script verwacht dat `locatie_scanner.py` NAAST het install script staat (in dezelfde map)
   - Kopieer ook `requirements.txt`

5. **Creeer venv en installeer dependencies**:
   ```powershell
   & $PythonExe -m venv "$InstallDir\venv"
   & "$InstallDir\venv\Scripts\pip.exe" install -r "$InstallDir\requirements.txt"
   ```

6. **Installeer/update NSSM service**:
   ```powershell
   & $NssmExe install $ServiceName "$InstallDir\venv\Scripts\python.exe" "$InstallDir\locatie_scanner.py"
   & $NssmExe set $ServiceName AppDirectory "$InstallDir"
   & $NssmExe set $ServiceName DisplayName "Locatie Scanner - Spinnekop"
   & $NssmExe set $ServiceName Description "Flask webapp voor barcode locatie scanning (poort $Port)"
   & $NssmExe set $ServiceName Start SERVICE_AUTO_START
   & $NssmExe set $ServiceName AppStdout "$InstallDir\logs\service.log"
   & $NssmExe set $ServiceName AppStderr "$InstallDir\logs\service.log"
   & $NssmExe set $ServiceName AppStdoutCreationDisposition 4
   & $NssmExe set $ServiceName AppStderrCreationDisposition 4
   & $NssmExe set $ServiceName AppRotateFiles 1
   & $NssmExe set $ServiceName AppRotateBytes 1048576
   & $NssmExe set $ServiceName AppRestartDelay 5000
   ```
   - `SERVICE_AUTO_START` = start bij boot (DEPL-02)
   - `AppRestartDelay 5000` = herstart na 5 sec bij crash (DEPL-02)
   - Log rotation op 1MB
   - `AppStdoutCreationDisposition 4` = append to existing log

7. **Creeer logs directory**: `New-Item -ItemType Directory -Force "$InstallDir\logs"`

8. **Start de service**: `& $NssmExe start $ServiceName`

9. **Verificatie**: Wacht 3 seconden, dan `Invoke-WebRequest http://localhost:$Port/api/ping` en toon resultaat. Als het faalt, toon `nssm status $ServiceName` output.

10. **Firewall rule** (idempotent):
    ```powershell
    $rule = Get-NetFirewallRule -DisplayName "Locatie Scanner" -ErrorAction SilentlyContinue
    if (-not $rule) {
        New-NetFirewallRule -DisplayName "Locatie Scanner" -Direction Inbound -Protocol TCP -LocalPort $Port -Action Allow
    }
    ```
    Opmerking in script: "Firewall rule bestaat waarschijnlijk al, dit is een safety check."
