Create `scripts/promote.ps1` following the established PowerShell patterns from build.ps1 and deploy.ps1.

**Script structure (6 steps):**

```
# Gebruik: powershell -ExecutionPolicy Bypass -File scripts/promote.ps1
$ErrorActionPreference = "Stop"
```

**Configuration block** (same variables as deploy.ps1):
- `$server = "10.0.1.5"`
- `$importShare = "\\$server\import"` (Z: drive = C:\import on server)
- `$importTestShare = "\\$server\import-test"` (Y: drive = C:\import-test on server)
- `$importDrive = "Z:"`
- `$importTestDrive = "Y:"`
- `$exeName = "bom-import-tool.exe"`

**Step 1: Versie uitlezen uit main.py** (same regex as build.ps1)
- Read `scripts/main.py` using `$PSScriptRoot` to find it
- Extract version via `__version__\s*=\s*'([^']+)'`
- Store in `$Version` variable
- Fail if pattern not found

**Step 2: Bevestiging vragen** (semi-auto safety net)
- Display: `"Promote versie $Version naar stable?"` in Cyan
- Display: `"  Dev:     Y:\bom-import-tool.exe -> Z:\bom-import-tool.exe"` in Gray
- Display: `"  Archief: Z:\bom-import-tool.exe -> Z:\archive\$Version\"` in Gray
- Prompt: `$confirm = Read-Host "Doorgaan? (j/n)"`
- Only proceed if `$confirm -eq 'j'` (case insensitive comparison with `-ieq`)
- On any other answer: `Write-Host "Afgebroken." -ForegroundColor Yellow` and `exit 0`

**Step 3: VPN verbinding controleren** (same pattern as deploy.ps1)
- `Test-Connection -ComputerName $server -Count 2 -Quiet`
- On failure: Dutch error message with VPN troubleshooting tips, `exit 1`

**Step 4: Drives mappen** (same idempotent pattern as deploy.ps1)
- Map Z: to `$importShare` if not already mapped (use `net use` check pattern from deploy.ps1)
- Map Y: to `$importTestShare` if not already mapped
- **Z: is REQUIRED here** (unlike deploy.ps1 where Z: was optional). Fail with clear error if Z: can't be mapped.
- Y: is also REQUIRED. Fail with clear error if Y: can't be mapped.

**Step 5: Archiveren + promoten** (the core operation)
- Check dev exe exists: `Test-Path "$importTestDrive\$exeName"` — fail if missing with message "Geen dev build gevonden op Y:\. Draai eerst build.ps1 en deploy.ps1"
- Define archive path: `$archiveDir = "$importDrive\archive\$Version"`
- Check if current stable exists: `Test-Path "$importDrive\$exeName"`
  - If YES: create archive dir `New-Item -ItemType Directory -Path $archiveDir -Force`, then `Copy-Item "$importDrive\$exeName" "$archiveDir\$exeName" -Force`, display "Huidige stable gearchiveerd naar Z:\archive\$Version\"
  - If NO: display "Geen bestaande stable gevonden — eerste promote" (this is fine for first-ever promote)
- Copy dev to stable: `Copy-Item "$importTestDrive\$exeName" "$importDrive\$exeName" -Force`

**Step 6: Verificatie**
- Verify stable exists: `Test-Path "$importDrive\$exeName"` — fail if not found
- Compare file sizes: `(Get-Item "$importDrive\$exeName").Length` should equal `(Get-Item "$importTestDrive\$exeName").Length`
- If sizes match: display "Promote geslaagd! v$Version staat live." in Green
- If sizes differ: display warning "WAARSCHUWING: Bestandsgrootte komt niet overeen — controleer handmatig" in Yellow
- Display final summary:
  - `"  Stable: C:\import\bom-import-tool.exe (v$Version)"` in Green
  - `"  Archief: C:\import\archive\$Version\"` in Green (only if archive was created)
  - `"  Evy's snelkoppeling werkt automatisch"` in Green

**Style notes:**
- All user-facing text in Dutch (matching deploy.ps1 and build.ps1)
- Use `Write-Host` with `-ForegroundColor` consistently (Cyan=headers, Yellow=steps/warnings, Green=success, Red=errors, Gray=info)
- Step numbering: `[1/6]`, `[2/6]`, etc.
