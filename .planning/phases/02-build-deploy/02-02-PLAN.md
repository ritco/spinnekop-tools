<task type="auto">
  <name>Task 1: Create deploy.ps1 — deploy exe to import-test on server</name>
  <files>scripts/deploy.ps1</files>
  <action>
Create `scripts/deploy.ps1` with these behaviors:

1. **Configuration block** at top of script:
   ```powershell
   $server = "10.0.1.5"
   $importShare = "\\$server\import"        # Z: drive — C:\import on server
   $importTestShare = "\\$server\import-test" # Y: drive — C:\import-test on server
   $importDrive = "Z:"
   $importTestDrive = "Y:"
   ```

2. **Pre-flight: find the exe to deploy**:
   - Look for `scripts/dist/bom-import-tool.exe` (relative to project root)
   - Also compute the scripts dir from `$PSScriptRoot`
   - If no exe found, print "Geen exe gevonden in dist/. Draai eerst build.ps1" and exit 1
   - Print which exe will be deployed and its file size

3. **Connectivity check**:
   - `Test-Connection -ComputerName $server -Count 2 -Quiet`
   - If fails: print "Server $server niet bereikbaar. Check VPN verbinding." and exit 1
   - If succeeds: print "Server bereikbaar via VPN"

4. **Map drives** (idempotent — skip if already mapped):
   - Check if Z: is mapped (`net use Z: 2>&1`). If not, map it: `net use Z: $importShare /persistent:no`
   - Check if Y: is mapped (`net use Y: 2>&1`). If not, map it: `net use Y: $importTestShare /persistent:no`
   - Use `/persistent:no` — these are session mappings, not permanent
   - If Y: mapping fails (share doesn't exist yet), print clear instructions:
     ```
     FOUT: Share \\10.0.1.5\import-test bestaat nog niet.
     Maak deze aan op de server via RDP:
       1. Open Computer Management > Shared Folders > Shares
       2. New Share > Path: C:\import-test
       3. Share name: import-test
       4. Permissions: Everyone = Full Control (intern netwerk)
     Daarna opnieuw proberen.
     ```
     Then exit 1.

5. **Copy exe to import-test**:
   - Source: `$PSScriptRoot\dist\bom-import-tool.exe`
   - Destination: `Y:\bom-import-tool.exe`
   - Use `Copy-Item -Force`
   - Print: "Deployed: bom-import-tool.exe -> C:\import-test\ op VMSERVERRUM"

6. **Verify copy**:
   - Check `Test-Path "Y:\bom-import-tool.exe"`
   - Print file size of deployed exe
   - If verification fails, exit 1

Script header:
```
# Deploy BOM Import Tool naar test-omgeving op VMSERVERRUM
# Gebruik: powershell -ExecutionPolicy Bypass -File scripts/deploy.ps1
# Vereist: VPN verbinding + import-test share op server
```

Error handling: `$ErrorActionPreference = "Stop"` at top. Wrap drive mapping in try/catch for clear error messages.
