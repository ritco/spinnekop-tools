  <action>
Create `scripts/build.ps1` with these behaviors:

1. **Read version from source**: Parse `scripts/main.py` for the line `__version__ = 'X.Y.Z'` using regex. Extract the version string. Fail with clear error if not found.

2. **Pre-flight checks**:
   - Verify `bom-import-tool.spec` exists in the scripts directory
   - Verify `pyinstaller` is available (test with `Get-Command pyinstaller`)
   - Print version being built: "Building bom-import-tool v{version}..."

3. **Run PyInstaller**:
   - Change to `scripts/` directory (PyInstaller spec uses relative paths)
   - Run: `pyinstaller --clean --noconfirm bom-import-tool.spec`
   - The `--clean` flag ensures no stale build artifacts
   - The `--noconfirm` flag prevents interactive prompts

4. **Version the output**:
   - PyInstaller produces `dist/bom-import-tool.exe` (name from spec)
   - Rename to `dist/bom-import-tool-{version}.exe`
   - Also keep a copy as `dist/bom-import-tool.exe` (for deploy script compatibility)
   - Print: "Build complete: dist/bom-import-tool-{version}.exe"

5. **Error handling**:
   - Use `$ErrorActionPreference = "Stop"`
   - If PyInstaller exits non-zero, print error and exit 1
   - If version parse fails, print "Could not find __version__ in main.py" and exit 1

Script header should include usage comment:
```
# Build BOM Import Tool
# Gebruik: powershell -ExecutionPolicy Bypass -File scripts/build.ps1
# Output: scripts/dist/bom-import-tool-{version}.exe
```

Do NOT modify the .spec file — it already works correctly. The build script wraps it.
