```python
@dataclass
class ConversionResult:
    errors: List[str] = field(default_factory=list)       # blokkerend
    warnings: List[str] = field(default_factory=list)     # niet-blokkerend
    matched: List[dict] = field(default_factory=list)     # {'bestelnummer', 'ridderiq_code', 'omschrijving', 'qty'}
    new_items: List[dict] = field(default_factory=list)   # {'bestelnummer', 'new_code', 'omschrijving', 'qty', 'fabrikant'}
    skipped: List[dict] = field(default_factory=list)     # {'rij', 'reden'}

    @property
    def has_blockers(self) -> bool:
        return len(self.errors) > 0
```

**Excel-parser `parse_eplan_excel(excel_path: str) -> tuple[str, List[EplanRow]]`:**

Vaste posities (beslissing uit CONTEXT.md):
- Rij 4 (index 3): projectnaam — lees cel A4 als projectnaam/stuklijstcode
- Rij 6 (index 5): kolomheaders — zoek kolommen op naam (niet positie)
- Rij 7+ (index 6+): dataregels

Kolommen zoeken op naam (case-insensitief, strip whitespace):
- `'Fabrikant'` → fabrikant
- `'Bestelnummer'` → bestelnummer
- `'Hoeveelheid'` → hoeveelheid
- `'Naam in schema'` → naam_in_schema

Bij kolom niet gevonden: raise `ValueError` met duidelijke melding inclusief gevonden headers.

Lege tussenrijen (alle relevante cellen leeg): stilzwijgend overslaan — geen fout, geen waarschuwing.

Extra (onbekende) kolommen: negeren.

Returns: `(projectnaam: str, rijen: List[EplanRow])`.

**Zoekterm-extractie `_extract_zoekterm(bestelnummer: str) -> tuple[str, bool]`:**

- Bevat punt: split op EERSTE punt, gebruik het deel NA het punt als zoekterm. Returns `(zoekterm, heeft_punt=True)`.
- Geen punt: gebruik volledige string als zoekterm. Returns `(bestelnummer, heeft_punt=False)`.

Voorbeelden:
- `'SE.GV2ME08'` → `('GV2ME08', True)`
- `'GV2ME08'` → `('GV2ME08', False)` — geeft ook een waarschuwing
- `'SE.LC1D09M7'` → `('LC1D09M7', True)`

**Artikel-code-generatie `get_next_26xxx_code(conn) -> str`:**

```sql
SELECT MAX(CODE) FROM R_ITEM WHERE CODE LIKE '26%'
```

Parse het resultaat: strip `'26'` prefix, converteer naar int, voeg 1 toe, formatteer terug als `'26{nummer:04d}'` (zodat codes minimaal 6 tekens zijn, bv. `'260001'` → `'260002'`).

Als query geen resultaat geeft of MAX is NULL: start bij `'260001'`.

**SQL-matching `match_components(rijen: List[EplanRow], conn) -> ConversionResult`:**

```
result = ConversionResult()

voor elke rij in rijen:
    zoekterm, heeft_punt = _extract_zoekterm(rij.bestelnummer)

    als niet heeft_punt:
        result.warnings.append(f"Rij {rij.excel_rij}: Bestelnummer '{rij.bestelnummer}' heeft geen punt — gebruik volledige string als zoekterm")

    sql = "SELECT CODE, DESCRIPTION FROM R_ITEM WHERE OMSCHRIJVING LIKE ?"
    zoek_param = f"%{zoekterm}%"
    cursor.execute(sql, (zoek_param,))
    matches = cursor.fetchall()

    als len(matches) == 0:
        new_code = get_next_26xxx_code(conn)
        # Verhoog lokale teller zodat volgende nieuwe code +1 is
        result.new_items.append({
            'bestelnummer': rij.bestelnummer,
            'new_code': new_code,
            'omschrijving': rij.naam_in_schema,
            'qty': rij.hoeveelheid,
            'fabrikant': rij.fabrikant,
        })

    als len(matches) == 1:
        result.matched.append({
            'bestelnummer': rij.bestelnummer,
            'ridderiq_code': matches[0].CODE,
            'omschrijving': matches[0].DESCRIPTION or rij.naam_in_schema,
            'qty': rij.hoeveelheid,
        })

    als len(matches) >= 2:
        conflict_codes = ', '.join(m.CODE for m in matches)
        result.errors.append(
            f"Rij {rij.excel_rij}: Bestelnummer '{rij.bestelnummer}' "
            f"heeft {len(matches)} matches in R_ITEM: {conflict_codes} — "
            f"pas OMSCHRIJVING aan of gebruik artikelcode direct"
        )
```

**BELANGRIJK voor nieuwe artikelcodes:** Houd een lokale teller bij zodat meerdere nieuwe artikelen in dezelfde run unieke codes krijgen. Haal MAX eenmalig op vóór de lus, verhoog de teller per nieuw artikel.

**Connectie-afhandeling in `match_components`:**

Ontvang `conn` als parameter (al geopend door de caller). Als `conn is None`, voeg een blokkerende fout toe: `"Geen verbinding met RidderIQ. Controleer VPN en serveradres."` en return vroeg.

**Hoofd-interface `convert(excel_path: str, env: str, dry_run: bool = True) -> ConversionResult`:**

```python
def convert(excel_path: str, env: str, dry_run: bool = True) -> ConversionResult:
    result = ConversionResult()

    # Stap 1: Excel parsen
    try:
        projectnaam, rijen = parse_eplan_excel(excel_path)
    except Exception as e:
        result.errors.append(f"Excel lezen mislukt: {e}")
        return result

    # Stap 2: Aggregeren
    rijen, skipped = aggregate_rows(rijen)
    result.skipped.extend(skipped)

    # Stap 3: SQL verbinding
    try:
        conn, info = get_connection(env)
    except Exception as e:
        result.errors.append(f"Verbindingsfout: {e}. Controleer VPN.")
        return result

    if conn is None:
        result.errors.append(f"Geen verbinding: {info}. Controleer VPN en serveradres.")
        return result

    # Stap 4: Matchen
    try:
        match_result = match_components(rijen, conn)
        result.errors.extend(match_result.errors)
        result.warnings.extend(match_result.warnings)
        result.matched.extend(match_result.matched)
        result.new_items.extend(match_result.new_items)
    finally:
        conn.close()

    # Stap 5: Stuklijst-duplicaatcheck (BOM-01)
    # Wordt in Plan 02 toegevoegd — hier nog niet

    # dry_run=True: geen bestanden schrijven
    # dry_run=False: delegeer aan write_output() (Plan 02)
    if not dry_run and not result.has_blockers:
        write_output(projectnaam, rijen, result, env)

    return result
```
