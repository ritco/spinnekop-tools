Twee wijzigingen in `scripts/locatie_scanner.py`:

**A) Self-signed SSL certificaat generatie + HTTPS serving**

1. Voeg `import ssl` en `import os` toe aan de imports (os is voor path checks).

2. Maak een functie `ensure_ssl_cert(cert_dir)` die:
   - `cert_dir` default naar de directory waar locatie_scanner.py staat (gebruik `os.path.dirname(os.path.abspath(__file__))`)
   - Checkt of `cert.pem` en `key.pem` bestaan in `cert_dir`
   - Als ze NIET bestaan: genereer een self-signed cert met Python's `ssl` stdlib — MAAR `ssl` kan geen certs genereren. Gebruik daarom het `cryptography` package:
     ```python
     from cryptography import x509
     from cryptography.x509.oid import NameOID
     from cryptography.hazmat.primitives import hashes, serialization
     from cryptography.hazmat.primitives.asymmetric import rsa
     import datetime
     ```
   - Genereer RSA 2048-bit key
   - Maak een self-signed cert met:
     - Subject/Issuer: CN=Locatie Scanner, O=Spinnekop
     - SAN (Subject Alternative Name): IP:10.0.1.5 en DNS:VMSERVERRUM en DNS:localhost — dit is KRITIEK voor Chrome, anders blijft de cert-waarschuwing terugkomen
     - Geldig voor 10 jaar (3650 dagen)
   - Schrijf `key.pem` en `cert.pem` naar `cert_dir`
   - Log met `log.info()` dat cert gegenereerd is
   - Return tuple `(cert_path, key_path)`
   - Als bestanden WEL bestaan: log "Using existing SSL cert" en return paden

3. Pas `if __name__ == "__main__"` aan:
   - Roep `ensure_ssl_cert()` aan voor `app.run()`
   - Gebruik `app.run(host="0.0.0.0", port=5050, debug=False, ssl_context=(cert_path, key_path))`
   - Pas de print-berichten aan: `https://` ipv `http://`
   - Log ook het cert-pad

4. Voeg `cryptography` toe aan requirements: update `scripts/deploy/locatie-scanner/requirements.txt` met `cryptography>=42.0.0` (geen exacte pin — de server kan niet downloaden, dus we installeren wat er op de wheel zit)

**B) Keyboard UX fix**

In de HTML/JS section van `locatie_scanner.py`, pas de `startScan()` functie aan:

1. Voeg als EERSTE regel in `startScan(target)` toe:
   ```javascript
   // Sluit virtueel toetsenbord door actief element te blurren
   if (document.activeElement) { document.activeElement.blur(); }
   ```
   Dit zorgt ervoor dat wanneer de gebruiker op het camera-icoontje tikt terwijl het input-veld actief is, het toetsenbord direct sluit voordat de camera-overlay opent.

NIET doen:
- Geen adhoc OpenSSL subprocess aanroepen — gebruik de cryptography Python library
- Geen aparte config voor cert-paden — conventie: cert.pem en key.pem naast het script
- Geen HTTP fallback — als cert generatie faalt, laat het crashen met duidelijke foutmelding
