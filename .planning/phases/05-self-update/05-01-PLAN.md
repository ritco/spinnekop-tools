  <action>
Drie wijzigingen in app_config.py:

**1. Timeout op UNC pad access in check_for_update():**
De huidige `version_file.exists()` call kan 30+ seconden hangen als de share onbereikbaar is. Fix dit door een threading-based timeout toe te voegen:
- Maak een helper `_read_version_json_with_timeout(share: str, timeout_sec: float = 2.0) -> dict | None`
- Gebruik `threading.Thread` + `threading.Event` om het bestandslezen in een aparte thread te doen
- Als de thread niet binnen `timeout_sec` klaar is, return `None` (geen update)
- Vang alle exceptions op in de thread (OSError, PermissionError, json.JSONDecodeError)
- Pas `check_for_update()` aan om deze helper te gebruiken ipv directe Path access
- Import `threading` bovenaan het bestand

**2. CTk update dialog als show_update_dialog():**
Voeg een nieuwe functie toe: `show_update_dialog(current_version: str, remote_version: str, parent=None) -> bool`
- Maak een `ctk.CTkToplevel` dialog (niet een plain messagebox)
- Toon titel "Update beschikbaar" in font_heading stijl
- Toon "Huidige versie: v{current}" en "Nieuwe versie: v{remote}" met duidelijke visuele hierarchie
- Twee knoppen: "Updaten" (groen, CLR_SUCCESS=#16A34A) en "Later" (grijs)
- Dialog is modal (grab_set)
- Return True als gebruiker op "Updaten" klikt, False als "Later" of venster gesloten
- Gebruik dezelfde font/kleur patronen als show_settings_dialog (font_heading, font_label, etc.)
- Import customtkinter BINNEN de functie (lazy import, net als show_settings_dialog)

**3. Fix _update.bat in do_self_update():**
De huidige batch heeft een bug: `ren "{new_exe.name}" "{local_exe.name}"` werkt alleen als de working directory de exe-directory is. Fix:
- Voeg `cd /d "{local_exe.parent}"` toe als eerste regel na `@echo off` en `timeout`
- Gebruik `move /y` ipv `del` + `ren` voor atomiciteit: `move /y "{new_exe.name}" "{local_exe.name}"` (dit overschrijft het doel als het bestaat, in 1 operatie)
- De huidige `del` + `ren` aanpak faalt als de oude exe nog gelockt is door het OS; `move /y` van .new naar de originele naam is robuuster
- Houd de `start ""` en `del "%~f0"` regels intact
