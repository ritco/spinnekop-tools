  <files>scripts/promote.ps1</files>
  <action>
Refactor het bestaande scripts/promote.ps1 met de volgende wijzigingen:

1. **Nieuwe parameter -Tool** (naast bestaande -Force):
   - Accepteert: "bom", "prod", of niets (standaard: beide tools)
   - Valideer dat de waarde "bom" of "prod" is, anders foutmelding en exit

2. **Bevestigingsprompt aanpassen** (stap 2/7):
   - Toon welke tool(s) gepromoot worden op basis van -Tool parameter
   - Als -Tool bom: toon alleen bom-import-tool info
   - Als -Tool prod: toon alleen productiestructuur info
   - Als geen -Tool: toon beide
   - -Force slaat bevestiging over (bestaand gedrag behouden)

3. **Promote logica herstructureren** (stap 5):
   - Maak een functie `Promote-Tool` die 1 tool promoot:
     - Parameters: $toolExeName, $toolVersion, $devDrive, $stableDrive, $archiveDir
     - Checkt of tool op Y: staat, zo niet: Write-Host waarschuwing en return $false
     - Archiveert huidige stable (als die bestaat)
     - Kopieert dev naar stable
     - Verifieert bestandsgrootte
     - Return $true bij succes
   - Roep Promote-Tool aan voor elke tool op basis van -Tool parameter
   - Track welke tools succesvol gepromoot zijn

4. **version.json archiveren** (voor het genereren van de nieuwe):
   - Als Z:\version.json bestaat: kopieer naar archiveDir VOOR promote
   - Dit gebeurt samen met de exe-archivering

5. **version.json atomair genereren** (NA alle exe-copies):
   - Lees bestaande version.json van Z: als die er is (voor de versie van de ANDERE tool)
   - Als version.json niet bestaat: maak nieuw aan met "0.0.0" voor de niet-gepromote tool
   - Overschrijf alleen de versie van de gepromote tool(s)
   - Schrijf version.json pas NA succesvolle exe-kopie(s)
   - Als geen enkele tool succesvol gepromoot: version.json NIET aanpassen

6. **Stap-nummering aanpassen**: van [N/6] naar [N/7] (extra stap voor version.json)

7. **Output samenvatting** (stap 7/7):
   - Toon welke tools gepromoot zijn met versienummers
   - Toon version.json inhoud
   - Toon archive locatie

Behoud alle bestaande patronen: $ErrorActionPreference = "Stop", kleurcodes (Yellow voor stappen, Green voor succes, Red voor fouten, Gray voor info), ping check, drive mapping, bestandsgrootte verificatie.
